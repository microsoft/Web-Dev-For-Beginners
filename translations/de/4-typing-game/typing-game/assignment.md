# Erstelle ein neues Tastaturspiel

## Anweisungen

Nachdem du die Grundlagen der ereignisgesteuerten Programmierung mit dem Tipp-Spiel gemeistert hast, ist es Zeit, deiner Kreativität freien Lauf zu lassen! Du wirst dein eigenes, tastaturbasiertes Spiel entwerfen und bauen, das dein Verständnis von Ereignisbehandlung, DOM-Manipulation und Benutzerinteraktionsmustern zeigt.

Erstelle ein kleines Spiel, das Tastaturereignisse nutzt, um bestimmte Aufgaben zu erfüllen. Dies könnte eine andere Art Tipp-Spiel sein, eine Kunst-Anwendung, die bei Tastendrücken Pixel auf dem Bildschirm malt, ein einfaches Arcade-Spiel, das mit den Pfeiltasten gesteuert wird, oder jedes andere kreative Konzept, das du dir vorstellen kannst. Werde kreativ und überlege, wie unterschiedliche Tasten unterschiedliche Verhaltensweisen auslösen können!

**Dein Spiel sollte beinhalten:**

| Anforderung | Beschreibung | Zweck |
|-------------|--------------|-------|
| **Ereignis-Listener** | Reagiere auf mindestens 3 verschiedene Tastaturereignisse | Demonstriere Verständnis der Ereignisbehandlung |
| **Visuelles Feedback** | Biete eine sofortige visuelle Rückmeldung auf Benutzereingaben | Zeige Beherrschung der DOM-Manipulation |
| **Spiel-Logik** | Integriere Punkte, Level oder Fortschrittsmechanismen | Übe die Implementierung des Anwendungszustands |
| **Benutzeroberfläche** | Klare Anweisungen und intuitive Steuerung | Entwickle Fähigkeiten im User Experience Design |

**Kreative Projektideen zum Nachdenken:**
- **Rhythmus-Spiel**: Spieler drücken Tasten im Takt zur Musik oder zu visuellen Hinweisen
- **Pixel-Art-Creator**: Verschiedene Tasten malen verschiedene Farben oder Muster
- **Wortbildner**: Spieler erstellen Wörter, indem sie Buchstaben in bestimmten Reihenfolgen tippen
- **Schlangenspiel**: Steuere eine Schlange mit den Pfeiltasten, um Gegenstände zu sammeln
- **Musik-Synthesizer**: Verschiedene Tasten spielen unterschiedliche musikalische Noten oder Klänge
- **Schnelltipp-Varianten**: Kategoriespezifisches Tippen (Programmierbegriffe, Fremdsprachen)
- **Keyboard-Drummer**: Erstelle Beats, indem Tasten verschiedenen Schlagzeugklängen zugeordnet werden

**Umsetzungstipps:**
- **Beginne** mit einem einfachen Konzept und erhöhe die Komplexität schrittweise
- **Setze Fokus** auf flüssige, reaktionsschnelle Steuerung, die sich natürlich anfühlt
- **Integriere** klare visuelle Anzeigen für Spielzustand und Spielerfortschritt
- **Teste** dein Spiel mit verschiedenen Nutzern, um intuitive Spielbarkeit sicherzustellen
- **Dokumentiere** deinen Code mit Kommentaren, die deine Ereignisbehandlungsstrategie erklären

## Bewertungsraster

| Kriterien | Hervorragend | Ausreichend | Verbesserungsbedürftig |
| --------- | ------------ | ----------- | ---------------------- |
| **Funktionalität** | Ein vollständiges, ausgereiftes Spiel mit mehreren Features und flüssigem Gameplay | Ein funktionierendes Spiel mit grundlegenden Features, das Tastaturereignisbehandlung zeigt | Eine minimale Umsetzung mit eingeschränkter Funktionalität oder erheblichen Fehlern |
| **Codequalität** | Gut organisierter, kommentierter Code nach Best Practices mit effizienter Ereignisbehandlung | Sauberer, lesbarer Code mit angemessenem Einsatz von Event Listeners und DOM-Manipulation | Grundlegende Code-Struktur mit einigen Organisationsproblemen oder ineffizienten Implementierungen |
| **Nutzererlebnis** | Intuitive Steuerung, klare Rückmeldungen und fesselndes Gameplay mit professionellem Gefühl | Funktionale Oberfläche mit ausreichender Benutzerführung und reaktiver Steuerung | Grundlegende Oberfläche mit unklaren Anweisungen oder schlechter Reaktionsfähigkeit |
| **Kreativität** | Originelles Konzept mit innovativem Einsatz von Tastaturereignissen und kreativer Problemlösung | Interessante Variante bekannter Spielmuster mit gutem Einsatz der Ereignisbehandlung | Einfache Umsetzung eines Basis-Konzepts mit minimalen kreativen Elementen |
