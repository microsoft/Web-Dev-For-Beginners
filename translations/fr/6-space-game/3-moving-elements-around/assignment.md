# Commentez Votre Code

## Instructions

Un code propre et bien documenté est essentiel pour maintenir et partager vos projets. Dans cet exercice, vous pratiquerez l'une des habitudes les plus importantes des développeurs professionnels : écrire des commentaires clairs et utiles qui expliquent le but et le fonctionnement de votre code.

Passez en revue votre fichier `app.js` actuel dans votre dossier de jeu, et trouvez des moyens de le commenter et de le ranger. Il est très facile que le code devienne difficile à gérer, et c’est une bonne occasion d'ajouter des commentaires pour vous assurer que votre code est lisible afin de pouvoir l'utiliser plus tard.

**Votre tâche inclut :**
- **Ajouter des commentaires** expliquant ce que fait chaque section majeure du code
- **Documenter les fonctions** avec des descriptions claires de leur objectif et de leurs paramètres
- **Organiser le code** en blocs logiques avec des en-têtes de section
- **Supprimer** tout code inutilisé ou redondant
- **Utiliser** des conventions de nommage cohérentes pour les variables et les fonctions

## Grille d’évaluation

| Critères | Exemplaire | Adéquat | Doit être amélioré |
| -------- | --------- | -------- | ----------------- |
| **Documentation du Code** | Le code `app.js` est entièrement commenté avec des explications claires et utiles pour toutes les sections et fonctions majeures | Le code `app.js` est suffisamment commenté avec des explications basiques pour la plupart des sections | Le code `app.js` a très peu de commentaires et manque d’explications claires |
| **Organisation du Code** | Le code est organisé en blocs logiques avec des en-têtes de section clairs et une structure cohérente | Le code a une certaine organisation avec un regroupement basique des fonctionnalités liées | Le code est quelque peu désorganisé et difficile à suivre |
| **Qualité du Code** | Toutes les variables et fonctions ont des noms descriptifs, aucun code inutilisé, suit des conventions cohérentes | La plupart du code suit de bonnes pratiques de nommage avec un minimum de code inutilisé | Les noms des variables ne sont pas clairs, contient du code inutilisé, style incohérent |
