# Praktyka na Typy Danych: Koszyk Zakupowy w E-commerce

## Instrukcje

Wyobraź sobie, że tworzysz nowoczesny system koszyka zakupowego w e-commerce. To zadanie pomoże Ci zrozumieć, jak różne typy danych w JavaScript współdziałają, by tworzyć aplikacje użyteczne w praktyce.

### Twoje Zadanie

Stwórz kompleksową analizę, jak wykorzystałbyś typy danych JavaScript w aplikacji koszyka zakupowego. Dla każdego z siedmiu prymitywnych typów danych oraz dla obiektów:

1. **Zidentyfikuj** typ danych i jego zastosowanie
2. **Wyjaśnij**, dlaczego ten typ danych jest najlepszym wyborem dla konkretnych funkcji koszyka
3. **Podaj** realistyczne przykłady kodu pokazujące użycie typu danych
4. **Opisz**, jak ten typ danych współpracuje z innymi elementami koszyka

### Wymagane Typy Danych do Omówienia

**Typy Prymitywne:**
- **String**: Nazwy produktów, opisy, informacje o użytkowniku
- **Number**: Ceny, ilości, obliczenia podatków
- **Boolean**: Dostępność produktów, preferencje użytkownika, status koszyka
- **Null**: Celowo puste wartości (np. brak kodów rabatowych)
- **Undefined**: Nieprzypisane wartości lub brakujące dane
- **Symbol**: Unikalne identyfikatory (zaawansowane użycie)
- **BigInt**: Duże operacje finansowe (zaawansowane użycie)

**Typy Referencyjne:**
- **Object**: Szczegóły produktów, profile użytkowników, zawartość koszyka
- **Array**: Lista produktów, historia zamówień, kategorie
