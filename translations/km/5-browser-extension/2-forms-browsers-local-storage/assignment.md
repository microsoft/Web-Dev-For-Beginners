# អនុម័ត API មួយ

## សេចក្ដីផ្តើម

APIs បើកឱកាសគ្មានដែនកំណត់សម្រាប់ការអភិវឌ្ឍបណ្ដាញវិស្ស័យបង្កើតច្នៃប្រឌិត! ក្នុងការប្រតិបត្តិការនេះ អ្នកនឹងជ្រើសរើស API ខាងក្រៅមួយ ហើយស្ថាបនាឧបករណ៍នៅក្នុងកម្មវិធីរុករកដែលដោះស្រាយបញ្ហាពិតប្រាកដ ឬផ្តល់នូវមុខងារដែលមានតម្លៃសម្រាប់អ្នកប្រើប្រាស់។

## សេចក្ដីណែនាំ

### ជំហានទី ១៖ ជ្រើសរើស API របស់អ្នក
ជ្រើសរើស API មួយពី [បញ្ជី API សាធារណៈឥតគិតថ្លៃដែលបានជ្រើសរើស](https://github.com/public-apis/public-apis)។ គិតពិចារណានូវប្រភេទដូចខាងក្រោម៖

**ជម្រើសពេញនិយមសម្រាប់អ្នកចាប់ផ្តើម៖**
- **កម្សាន្ត**៖ [Dog CEO API](https://dog.ceo/dog-api/) សម្រាប់រូបថតសត្វឆ្កែបែបចៃដន្យ
- **អាកាសធាតុ**៖ [OpenWeatherMap](https://openweathermap.org/api) សម្រាប់ទិន្នន័យអាកាសធាតុបច្ចុប្បន្ន
- **ខួបសុន្ទរកថា**៖ [Quotable API](https://quotable.io/) សម្រាប់សុន្ទរកថាចម្លែក
- **ព័ត៌មាន**៖ [NewsAPI](https://newsapi.org/) សម្រាប់ចំណងជើងព័ត៌មានបច្ចុប្បន្ន
- **ហេតុការណ៍រីករាយ**៖ [Numbers API](http://numbersapi.com/) សម្រាប់ព័ត៌មានលម្អិតពេលលេខ

### ជំហានទី ២៖ ធ្វើផែនការអំពីកម្មវិធីបន្ថែមរបស់អ្នក
មុននឹងសរសេរកូដ សូមឆ្លើយសំណួរផ្ទះបាយខាងក្រោម៖
- កម្មវិធីបន្ថែមរបស់អ្នកដោះស្រាយបញ្ហាអ្វី?
- អ្នកប្រើផ្ដោតលើនរណា?
- តើអ្នកនឹងរក្សាទិន្នន័យអ្វីនៅក្នុង local storage?
- តើអ្នកនឹងដោះស្រាយបញ្ហាពេល API ខូច ឬកំណត់ចំនួនកម្ដៅយ៉ាងដូចម្តេច?

### ជំហានទី ៣៖ ស្ថាបនាកម្មវិធីបន្ថែមរបស់អ្នក
កម្មវិធីបន្ថែមរបស់អ្នកគួរត្រូវមាន៖

**លក្ខណៈជាមូលដ្ឋាន៖**
- បង្កើតទម្រង់បញ្ចូលសម្រាប់ប៉ារ៉ាម៉ែត្រ API ដែលត្រូវការ
- បញ្ចូល API ដែលមានការដោះស្រាយកំហុសយ៉ាងត្រឹមត្រូវ
- ការកំណត់ local storage សម្រាប់ចំណង់ចំណូលចិត្តអ្នកប្រើ ឬក្បាល API
- មុខងារជាមួយ UI ដែលស្អាត និងរហ័សចម្លើយ
- ការបង្ហាញស្ថានភាពកំពុងរុំរឿង និងមតិពីអ្នកប្រើ

**តម្រូវការកូដ៖**
- ប្រើមុខងារ JavaScript នាទីសព្វថ្ងៃ (ES6+)
- អនុវត្ត async/await សម្រាប់ការហៅ API
- រួមបញ្ចូលការដោះស្រាយកំហុសត្រឹមត្រូវជាមួយ try/catch
- បន្ថែមកំណត់សំគាល់មានន័យបញ្ជាក់កូដរបស់អ្នក
- អនុវត្តតាមទម្រង់កូដដោយមានរបៀបអនុវត្តត្រឹមត្រូវ

### ជំហានទី ៤៖ បំពេញការសាកល្បង និងសម្អាត
- សាកល្បងកម្មវិធីបន្ថែមជាមួយបញ្ចូលនានា
- ដោះស្រាយករណីចុងដោយមិនមានអ៊ីនធឺណិត ឬចម្លើយ API មិនត្រឹមត្រូវ
- ធានាថាកម្មវិធីបន្ថែមដំណើរការបន្តបន្ទាប់ពីបើកកម្មវិធីរុករកឡើងវិញ
- បន្ថែមសារ​កំហុស​ងាយ​ស្រួល​សម្រាប់​អ្នកប្រើ

## មុខងារបន្ថែម

នាំកម្មវិធីបន្ថែមរបស់អ្នកទៅកម្រិតបន្ទាប់៖
- បន្ថែមចំនុចចូល API ច្រើនក្នុងមុខងារ​មានតម្លៃជាងមុន
- អនុវត្តការផ្ទុកទិន្នន័យដើម្បីកាត់បន្ថយការហៅ API
- បង្កើតខ្សែក្ចាត់ក្តារចុចសម្រាប់សកម្មភាពទូទៅ
- បន្ថែមមុខងារនាំចេញ/នាំចូលទិន្នន័យ
- អនុវត្តជម្រើសប្តូរតាមបំណងអ្នកប្រើ

## តម្រូវការដាក់ស្នើ

1. **កម្មវិធីបន្ថែមក្នុងកម្មវិធីរុករកដែលដំណើរការបាន** ដែលបញ្ចូលជោគជ័យជាមួយ API ដែលបានជ្រើសរើស
2. **ឯកសារ README** អធិប្បាយ៖
   - API ដែលអ្នកបានជ្រើសរើស និងហេតុផល
   - របៀបដំឡើង និងប្រើកម្មវិធីបន្ថែមរបស់អ្នក
   - ក្បាល API ឬការកំណត់ផ្សេងៗ
   - រូបថតអេក្រង់នៃកម្មវិធីបន្ថែមការជាក់ស្តែង
3. **កូដស្អាត មានកំណត់សំគាល់** និងអនុវត្ត JavaScript ទំនើប

## លំហាត់ចាប់ផ្តើម

1. **ចាប់ផ្តើមពីសាមញ្ញ**៖ ជ្រើស API មិនត្រូវការសៀងសៀងស្នូលសុវត្ថិភាពច្រើន
2. **អានឯកសារ**៖ យល់ដឹងយ៉ាងល្អអំពីចំណុចចូល និងចម្លើយ API ដែលបានជ្រើសរើស
3. **ផែនការអតិថិជន UI**៖ គូរ និងរៀបចំ UI របស់កម្មវិធីបន្ថែមមុនសរសេរកូដ
4. **សាកល្បងជានិច្ច**៖ បន្ថែមមុខងារធ្វើការសាកល្បងរៀងៗខ្លួន
5. **ដោះស្រាយកំហុស**៖ ស្រមៃថាការហៅ API អាចបរាជ័យ ហើយផែនការយ៉ាងត្រឹមត្រូវ
