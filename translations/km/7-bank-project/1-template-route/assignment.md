# ធ្វើឱ្យការបញ្ជូនទំនិញល្អប្រសើរឡើង

## សេចក្តីណែនាំ

ឥឡូវនេះដែលអ្នកបានបង្កើតប្រព័ន្ធបញ្ជូនទំនិញមូលដ្ឋានរួចរាល់ហើយ ពេលវេលានេះគឺដើម្បីបន្ថែមមុខងារដែលមានវិជ្ជាជីវៈ ដែលធ្វើឱ្យបទពិសោធន៍អ្នកប្រើកាន់តែល្អ និងផ្តល់ឧបករណ៍ល្អសម្រាប់អ្នកអភិវឌ្ឍន៍។ កម្មវិធីពិតប្រាកដត្រូវការច្រើនជាងការប្ដូរផ្ទាំងគ្រាន់តែប៉ុណ្ណោះ – ពីមុខគេត្រូវការចំណងជើងបណ្ដាញផ្ទាល់ដែលផ្លាស់ប្តូរ, សោប្រើសម្រាប់ជីវិតនៃទំព័រ និងស្ថាបត្យកម្មដែលអាចពង្រីកបាន។

ក្នុងការងារនេះ អ្នកនឹងធ្វើបន្ថែមលើការអនុវត្តរបស់អ្នកទាក់ទងនឹងបញ្ជូនទ្រង់ទ្រាយ ជាមួយមុខងារសំខាន់២ចំនុច ដែលឃើញបានជាញឹកញាប់ក្នុងកម្មវិធីបណ្ដាញបញ្ចេញលទ្ធផល។ ការកែលម្អទាំងនេះនឹងធ្វើឲ្យកម្មវិធីធនាគាររបស់អ្នកមានភាពស្អាតល្អប្រសើរឡើង ហើយផ្តល់មូលដ្ឋានសម្រាប់មុខងារពេលខាងមុខ។

ការបញ្ជាក់នៃគន្លងទំនិញឥឡូវនេះ មានត្រឹមតែរបស់ផ្ទាំងរបស់ទំព័រតែមួយដែលត្រូវប្រើប្រាស់។ ប៉ុន្តែពេលបង្ហាញទំព័រថ្មី នោះត្រូវការផ្នែកបន្ថែមប្រែប្រួលបន្តិច។ ចូរធ្វើឲ្យកម្មវិធីរបស់យើងមានមុខងារបន្ថែមពីរខាងក្រោម៖

### មុខងារ ១៖ ចំណងជើងទំព័រដោយមានលក្ខណៈបម្លែងតាមឈុត
**គោលបំណង៖** ផ្តល់ចំណងជើងសម្រាប់គ្រប់ផ្ទាំង និងធ្វើឲ្យចំណងជើងផ្ទាំងបង្ហាញលើកញ្ចក់ត្រូវបំលែងតាមចំណងជើងថ្មីពេលផ្ទាំងផ្លាស់ប្តូរ។

**ហេតុអ្វីបានជា វាមានសារៈសំខាន់៖**
- **ធ្វើឲ្យ**បទពិសោធន៍អ្នកប្រើកាន់តែប្រសើរ ដោយបង្ហាញចំណងជើងផ្ទាំងកម្មវិធីដោយពណ៌នាបានច្បាស់លាស់
- **បង្កើន**ការចូលដល់សម្រាប់អ្នកដែលប្រើអ្នកអានអេក្រង់ និងបច្ចេកវិទ្យាជំនួយ
- **ផ្តល់**ការកត់សម្គាល់ល្អជាងក្នុងបណ្ណាល័យវេបប្រាក់ហ្វ្រេម និងប្រវត្តិរបស់កម្មវិធីកម្មវិធីរុករក
- **អនុលោម**ទៅវិជ្ជាជីវៈល្អបំផុតនៃការអភិវឌ្ឍន៍វេប

**វិធីសាស្រ្តអនុវត្ត៖**
- **បន្ថែម**វត្ថុ routes ដើម្បីបញ្ចូលព័ត៌មានចំណងជើងសម្រាប់គ្រប់គន្លង
- **កែប្រែ**មុខងារ `updateRoute()` ដើម្បីធ្វើឲ្យ `document.title` ផ្លាស់ប្តូរយ៉ាង δυναμικός
- **សាកល្បង**ថាចំណងជើងផ្លាស់ប្តូរយ៉ាងត្រឹមត្រូវពេលបន្តផ្ទាំងក្នុងមួយ

### មុខងារ ២៖ សោជីវិតរបស់គន្លង  
**គោលបំណង៖** បន្ថែមជម្រើសដំណើរការកូដមួយបន្ទាប់ពីផ្ទាំងផ្លាស់ប្តូរ។ យើងចង់បញ្ចេញ `'Dashboard is shown'` នៅក្នុងកុងសូលអ្នកអភិវឌ្ឍន៍រាល់ពេលផ្ទាំងផ្ទាំងគ្រប់គ្រងត្រូវបង្ហាញ។

**ហេតុអ្វីបានជា វាមានសារៈសំខាន់៖**
- **អនុញ្ញាត**ការប្រតិបត្តិភាសារប្រព័ន្ធប្ដូរកូដពិសេសពេលគន្លងពិសេសត្រូវផ្ទុក
- **ផ្តល់**សោសម្រាប់ការវិភាគ, ការចុះបញ្ជីឬកូដចាប់ផ្តើម
- **បង្កើត**មូលដ្ឋានសម្រាប់សកម្មភាពគន្លងកាន់តែស្មុគស្មាញ
- **បង្ហាញ**តួអង្គគំរូអ្នកគ្រប់គ្រងនៅក្នុងការអភិវឌ្ឍន៍វេប

**វិធីសាស្រ្តអនុវត្ត៖**
- **បន្ថែម**គុណលក្ខណៈមុខងារហៅត្រឡប់ជាជម្រើសក្នុងការកំណត់គន្លង
- **ដំណើរការ**មុខងារហៅត្រឡប់ (បើមាន) បន្ទាប់ពីការបង្ហាញផ្ទាំងបានបញ្ចប់
- **ធានា**មុខងារ អាចប្រើបានសម្រាប់គន្លងណាមួយដែលកំណត់ហៅត្រឡប់
- **សាកល្បង**ថាសារ console បង្ហាញពេលមានការចូលទៅផ្ទាំងគ្រប់គ្រង
