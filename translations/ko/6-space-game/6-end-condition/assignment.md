# 샘플 게임 만들기

## 과제 개요

이제 우주 게임에서 게임 종료 조건과 재시작 기능을 마스터했으니, 이 개념들을 완전히 새로운 게임 경험에 적용할 차례입니다. 다양한 종료 조건 패턴과 재시작 메커니즘을 보여주는 자신만의 게임을 설계하고 구축하세요.

이 과제는 게임 디자인에 대해 창의적으로 사고하는 동시에 배운 기술적 역량을 연습하도록 도전합니다. 다양한 승리 및 패배 시나리오를 탐구하고, 플레이어 진행을 구현하며, 흥미로운 재시작 경험을 만듭니다.

## 프로젝트 요구사항

### 핵심 게임 기능

게임에는 다음과 같은 필수 요소가 포함되어야 합니다:

**종료 조건 다양성**: 게임이 끝나는 최소 두 가지 방식을 구현하세요:
- **점수 기반 승리**: 플레이어가 목표 점수에 도달하거나 특정 아이템을 수집
- **목숨 기반 패배**: 플레이어가 모든 목숨 또는 체력을 잃음
- **목표 달성**: 모든 적 처치, 특정 아이템 수집 또는 목표 달성
- **시간 기반**: 설정된 시간 또는 카운트다운이 0이 되면 종료

**재시작 기능**:
- **게임 상태 초기화**: 이전 게임 객체 모두 제거 및 변수 초기화
- **시스템 재설정**: 새로운 플레이어 상태, 적, 목표와 함께 새로 시작
- **사용자 친화적 조작법**: 게임 재시작을 위한 명확한 안내 제공

**플레이어 피드백**:
- **승리 메시지**: 플레이어 성취를 축하하는 긍정적인 피드백
- **패배 메시지**: 재도전을 독려하는 용기 부여 메시지 제공
- **진행 표시기**: 현재 점수, 목숨, 목표 상태 표시

### 게임 아이디어 및 영감

아래 게임 컨셉 중 하나를 선택하거나 자신만의 게임을 만드세요:

#### 1. 콘솔 어드벤처 게임
전투 메커니즘이 포함된 텍스트 기반 모험 게임 만들기:

```
Hero> Strikes with broadsword - orc takes 3p damage
Orc> Hits with club - hero takes 2p damage  
Hero> Kicks - orc takes 1p damage
Game> Orc is defeated - Hero collects 2 coins
Game> ****No more monsters, you have conquered the evil fortress****
```

**구현할 핵심 기능:**
- 다양한 공격 옵션이 있는 **턴제 전투**
- 플레이어와 적 모두의 **체력**
- 동전이나 아이템을 수집하는 **인벤토리 시스템**
- 난이도가 다른 **여러 유형의 적**
- 모든 적 처치 시 **승리 조건**

#### 2. 수집 게임
- **목표**: 장애물을 피하면서 특정 아이템 수집
- **종료 조건**: 목표 수집 횟수 달성하거나 목숨 모두 소진
- **진행**: 게임이 진행될수록 아이템 접근 난이도 증가

#### 3. 퍼즐 게임
- **목표**: 점차 어려워지는 퍼즐 해결
- **종료 조건**: 모든 레벨 완료 또는 이동/시간 소진
- **재시작**: 진행 상황 초기화 후 첫 레벨로 초기화

#### 4. 디펜스 게임
- **목표**: 적의 웨이브로부터 기지 보호
- **종료 조건**: 모든 웨이브 생존(승리) 또는 기지 파괴(패배)
- **진행**: 적 웨이브 난이도 및 수 증가

## 구현 지침

### 시작하기

1. **게임 디자인 계획**:
   - 기본 게임 플레이 루프 스케치
   - 종료 조건 명확히 정의
   - 재시작 시 리셋할 데이터 식별

2. **프로젝트 구조 설정**:
   ```
   my-game/
   ├── index.html
   ├── style.css
   ├── game.js
   └── README.md
   ```

3. **핵심 게임 루프 작성**:
   - 게임 상태 초기화
   - 사용자 입력 처리
   - 게임 로직 업데이트
   - 종료 조건 확인
   - 현재 상태 렌더링

### 기술적 요구사항

**최신 자바스크립트 사용**:
- 변수 선언에 `const`와 `let` 사용
- 적절한 곳에 화살표 함수 활용
- 템플릿 리터럴, 구조 분해 할당 등 ES6+ 기능 적용

**이벤트 기반 아키텍처**:
- 사용자 상호작용에 대한 이벤트 핸들러 생성
- 이벤트를 통한 게임 상태 변경 구현
- 재시작 기능에 이벤트 리스너 활용

**깔끔한 코드 관행**:
- 단일 책임 함수 작성
- 설명적인 변수 및 함수 이름 사용
- 게임 로직과 규칙에 대한 주석 추가
- 논리적 구역으로 코드 구성
