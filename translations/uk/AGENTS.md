# AGENTS.md

## Огляд проекту

Це навчальний репозиторій для викладання основ веб-розробки початківцям. Навчальний курс — це комплексний 12-тижневий курс, розроблений Microsoft Cloud Advocates, який містить 24 практичні уроки, що охоплюють JavaScript, CSS і HTML.

### Ключові компоненти

- **Навчальний контент**: 24 структуровані уроки, організовані у проектні модулі
- **Практичні проекти**: Терраріум, Гра на друкування, Розширення браузера, Космічна гра, Банківський додаток, Редактор коду та AI чат-асистент
- **Інтерактивні тести**: 48 тестів по 3 запитання в кожному (до і після уроку)
- **Підтримка кількох мов**: Автоматичні переклади більш ніж 50 мовами за допомогою GitHub Actions
- **Технології**: HTML, CSS, JavaScript, Vue.js 3, Vite, Node.js, Express, Python (для AI-проектів)

### Архітектура

- Навчальний репозиторій зі структурою на основі уроків
- Кожна папка уроку містить README, приклади коду та рішення
- Окремі проекти у власних директоріях (quiz-app, різні проекти уроків)
- Система перекладів через GitHub Actions (co-op-translator)
- Документація подається за допомогою Docsify та доступна у форматі PDF

## Команди налаштування

Цей репозиторій призначений головним чином для споживання навчального контенту. Для роботи з конкретними проектами:

### Основне налаштування репозиторію

```bash
git clone https://github.com/microsoft/Web-Dev-For-Beginners.git
cd Web-Dev-For-Beginners
```

### Налаштування Quiz App (Vue 3 + Vite)

```bash
cd quiz-app
npm install
npm run dev        # Запустити сервер розробки
npm run build      # Збірка для продакшену
npm run lint       # Запустити ESLint
```

### API проекту Bank (Node.js + Express)

```bash
cd 7-bank-project/api
npm install
npm start          # Запустити API сервер
npm run lint       # Запустити ESLint
npm run format     # Відформатувати за допомогою Prettier
```

### Проекти розширень браузера

```bash
cd 5-browser-extension/solution
npm install
# Дотримуйтесь інструкцій завантаження розширень, специфічних для браузера
```

### Проекти космічної гри

```bash
cd 6-space-game/solution
npm install
# Відкрийте index.html у браузері або використайте Live Server
```

### Проект чату (Python Backend)

```bash
cd 9-chat-project/solution/backend/python
pip install openai
# Встановіть змінну оточення GITHUB_TOKEN
python api.py
```

## Робочий процес розробки

### Для авторів контенту

1. **Форкніть репозиторій** в свій акаунт GitHub
2. **Клонувати форк** локально
3. **Створіть нову гілку** для своїх змін
4. Вносьте зміни в контент уроків або приклади коду
5. Тестуйте всі зміни коду у відповідних директоріях проектів
6. Надсилайте pull-реквести, дотримуючись правил внесків

### Для учнів

1. Форкніть або клонувати репозиторій
2. Перейдіть послідовно до директорій уроків
3. Читайте README файли кожного уроку
4. Виконуйте тести перед уроком на https://ff-quizzes.netlify.app/web/
5. Працюйте з прикладами коду у папках уроків
6. Виконуйте завдання і виклики
7. Пройдіть тести після уроку

### Жива розробка

- **Документація**: Запуск `docsify serve` в корені (порт 3000)
- **Quiz App**: Запуск `npm run dev` в каталозі quiz-app
- **Проекти**: Використовуйте розширення VS Code Live Server для HTML-проектів
- **API-проекти**: Запуск `npm start` у відповідних директоріях API

## Система перекладу

### Автоматичний переклад

- Використовує GitHub Actions з workflow co-op-translator
- Перекладає автоматично на понад 50 мов
- Файли джерел у основних каталогах
- Перекладені файли в `translations/{language-code}/`

### Додавання ручних покращень перекладів

1. Знайдіть файл у `translations/{language-code}/`
2. Внесіть покращення, зберігаючи структуру
3. Переконайтеся, що приклади коду залишаються працездатними
4. Тестуйте будь-який локалізований контент тестів

### Метадані перекладу

Перекладені файли містять заголовок із метаданими:
```markdown

  "original_hash": "...",

  "source_file": "...",
  "language_code": "..."

-->
```

## Налагодження та усунення несправностей

### Поширені проблеми

**Quiz app не запускається**:
- Перевірте версію Node.js (рекомендується v14+)
- Видаліть `node_modules` та `package-lock.json`, потім запустіть `npm install` знову
- Перевірте конфлікти портів (за замовчуванням Vite використовує порт 5173)

**API сервер не запускається**:
- Переконайтеся, що версія Node.js відповідає мінімальній (node >=10)
- Перевірте, чи порт не зайнятий
- Переконайтеся, що всі залежності встановлені через `npm install`

**Розширення браузера не завантажується**:
- Перевірте правильність форматування manifest.json
- Перегляньте консоль браузера на помилки
- Дотримуйтесь інструкцій з встановлення для конкретного браузера

**Проблеми з Python чат-проектом**:
- Впевніться, що пакет OpenAI встановлений: `pip install openai`
- Переконайтеся, що встановлено змінну оточення GITHUB_TOKEN
- Перевірте права доступу до GitHub Models

**Docsify не подає документацію**:
- Встановіть docsify-cli глобально: `npm install -g docsify-cli`
- Запускайте з кореневого каталогу репозиторію
- Переконайтеся, що існує `docs/_sidebar.md`

### Поради щодо середовища розробки

- Використовуйте VS Code з розширенням Live Server для HTML-проектів
- Встановіть розширення ESLint і Prettier для послідовного форматування
- Використовуйте DevTools браузера для налагодження JavaScript
- Для проектів Vue встановіть розширення Vue DevTools для браузера

### Особливості продуктивності

- Велика кількість перекладених файлів (понад 50 мов) означає, що повні клони великі
- Використовуйте shallow clone, якщо працюєте лише з контентом: `git clone --depth 1`
- Ігноруйте переклади при пошуку в англомовному контенті
- Процеси збірки можуть бути повільними при першому запуску (npm install, Vite build)

## Міркування з безпеки

### Змінні оточення

- API ключі ніколи не повинні комітися в репозиторій
- Використовуйте `.env` файли (вже є у `.gitignore`)
- Документуйте необхідні змінні оточення у README проектів

### Python-проекти

- Використовуйте віртуальні оточення: `python -m venv venv`
- Підтримуйте залежності оновленими
- Токени GitHub повинні мати мінімально необхідні права

### Доступ до GitHub Models

- Потрібні Персональні Токени Доступу (PAT) для GitHub Models
- Токени зберігайте як змінні оточення
- Ніколи не комітьте токени або облікові дані
