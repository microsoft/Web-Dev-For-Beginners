# Data Types Practice: Carrinho de Compras de Comércio Eletrónico

## Instruções

Imagine que está a construir um sistema moderno de carrinho de compras para comércio eletrónico. Este exercício irá ajudá-lo a compreender como diferentes tipos de dados JavaScript funcionam em conjunto para criar aplicações do mundo real.

### A Sua Tarefa

Crie uma análise abrangente sobre como usaria os tipos de dados JavaScript numa aplicação de carrinho de compras. Para cada um dos sete tipos de dados primitivos e objetos, necessita de:

1. **Identificar** o tipo de dado e o seu propósito
2. **Explicar** por que este tipo de dado é a melhor escolha para funcionalidades específicas do carrinho de compras
3. **Fornecer** exemplos realistas de código que mostrem o tipo de dado em ação
4. **Descrever** como este tipo de dado interage com outras partes do carrinho de compras

### Tipos de Dados Obrigatórios a Cobrir

**Tipos de Dados Primitivos:**
- **String**: Nomes de produtos, descrições, informação do utilizador
- **Number**: Preços, quantidades, cálculos de impostos
- **Boolean**: Disponibilidade de artigos, preferências do utilizador, estado do carrinho
- **Null**: Valores intencionalmente vazios (como códigos de desconto em falta)
- **Undefined**: Valores não inicializados ou dados em falta
- **Symbol**: Identificadores únicos (uso avançado)
- **BigInt**: Cálculos financeiros de grande dimensão (uso avançado)

**Tipos de Referência:**
- **Object**: Detalhes do produto, perfis de utilizador, conteúdos do carrinho
- **Array**: Lista de produtos, histórico de encomendas, categorias
